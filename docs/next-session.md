> 作成: 2026-03-22（タスク B・A・C 完了後）

---

## このドキュメントの使い方

このセッションでは **Claude（Claude Code ではなく、会話形式）** と以下の流れで進めてください。

1. タスクごとに **Claude に相談して方針を決める**
2. 方針が固まったら **Claude Code 向けの指示文を作ってもらう**
3. Claude Code で実装 → 完了したら次のタスクへ

実装の細かい判断は Claude Code に任せる方針です。仕様・制約・完了条件を中心に指示文を書いてもらうよう伝えてください。

## 残りタスク一覧と進め方

### タスク D：Activity ごとの統計追加（平均間隔）

**概要**：`/activities/[activityId]` の統計に「平均間隔」を追加する。

> **注記**：`server/queries/activity.ts` を確認すると `avgIntervalDays` はすでにクエリ側で計算済み、`activity-detail.tsx` でも「だいたいの間隔」として表示済みの可能性がある。次セッション開始時に実際の画面で確認してから着手すること。表示されていれば D は完了済みとして扱ってよい。

**相談すること**（未実装だった場合）：
- 平均間隔の定義：全期間の平均か、直近 N 件の平均か
- 表示形式：「平均 X 日おき」テキストか、グラフか
- 他に追加したい指標があるか（月別記録数グラフ、連続記録日数など）

### タスク E：peaklog 記録は軽くの方向性（設計相談）

**概要**：2日後・3日後、または1日が終わってからまとめて記録するパターンが多い。ログ作成の導線について検討したい。

**相談すること**：
- 現状のクイックログ（Home の Activity タップ → モーダル）は「その瞬間に記録する」想定。まとめ記録のユースケースに対応するために何か変える必要があるか
- History ページから直接「この日にログを追加」できる導線を作るか、または現状の日時選択（「他の日」カレンダー）で十分とするか

## Claude Code 指示文の作り方（前セッションの流れ）

### 流れ

**1. Claude にリポジトリのコードを確認してもらう**
- 「〇〇について相談したい」と伝えれば Claude が `okmr2217/peak-log` から該当ファイルを自分で取得してくれる

**2. 相談して方針を決める**
- 選択肢がある場合は Claude が案を提示 → どれにするか選ぶ
- 迷ったら「それぞれのメリット・デメリットを教えて」と聞けばよい
- 「おすすめで」と言えば Claude が判断してくれる

**3. 指示文を作ってもらう**
- 方針が決まったら「指示文を作成して」と伝える
- 仕様・制約・完了条件・session-log 更新手順・コミットメッセージが含まれる形式で作成される
- 実装コストが小さいタスクは束ねて 1 セッションで実施することも可能

**4. Claude Code で実装**
- 指示文をそのまま Claude Code に渡す
- 実装完了後、session-log と handoff.md の更新もセットで行う（指示文に含まれている）

**5. 次のタスクへ**
- 完了を確認したら「〇〇は完了しました。次は〇〇について相談したい」と伝えて続ける
